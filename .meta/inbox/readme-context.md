# README Context Notes

From Phil (2026-04-30), during philset v0.2.1 session:

The README needs to be written for humans, not agents. Early drafts
will likely read like agent-facing documentation — explaining concepts
in terms of how Claude should use them, rather than explaining what
philset IS and why a developer would want it.

The iterative /draft flow is designed to catch exactly this kind of
drift. The README will need a few rounds.

Key framing for the README: philset's assess → draft → implement flow
is essentially a slower, more conversational /plan implementation
designed to minimize rework. It supersedes Claude Code's built-in plan
mode for philset users — the design doc IS the plan, but it's
collaborative and produces a durable artifact.
